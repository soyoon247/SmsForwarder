# SMS 자동 전달 앱 설계 — 보완 및 수정 사항

본 문서는 기존 설계의 안정성, Android 정책 대응, 유지보수성 향상을 위한 보완 사항을 정리한다.

---

## 1. Android 동작 제약 및 정책 대응

### 기본 SMS 앱 여부 관련 제한

Android 10 이상에서 SMS 관련 권한을 사용하는 앱은 일부 제조사 환경에서 백그라운드 동작 제한이 발생할 수 있다.

- 기본 SMS 앱이 아닌 경우
  - SMS 수신은 가능하나 전달 동작이 제한될 수 있음
  - 기기 제조사 정책에 따라 동작 불안정 가능
- 사용자 안내 필요
  - 일부 기기에서 정상 동작하지 않을 수 있음
  - 필요 시 기본 SMS 앱 설정을 안내

---

## 2. Foreground Service 구조 단순화

현재 설계에서는 SMS 수신을 BroadcastReceiver에서 직접 처리하므로 장시간 실행 서비스가 필요하지 않다.

### 수정 사항

- Foreground Service 제거
- BootReceiver 제거
- SMS 처리 책임을 SmsReceiver 단일 컴포넌트로 유지

### 기대 효과

- 배터리 정책 충돌 감소
- 권한 요구 최소화
- 시스템 안정성 향상

---

## 3. 저장 구조 개선

SharedPreferences + JSON 방식은 로그 증가 시 성능 저하 가능성이 있다.

### 필수 보완

- 전달 로그 최대 개수 제한
  - 권장: 100개
  - 초과 시 오래된 로그 삭제

### 확장 시 권장 구조

- Room Database 사용
  - ForwardRule 테이블
  - ForwardLog 테이블

---

## 4. 전화번호 정규화 처리

SMS 발신 번호는 다양한 형식으로 수신될 수 있다.

### 예시

- +821012345678
- 010-1234-5678
- 01012345678

### 필수 처리

- 공백 제거
- 하이픈 제거
- 국가번호 +82 → 0 변환

정규화된 번호 기준으로 규칙 매칭 수행

---

## 5. Multipart SMS 처리

긴 SMS는 여러 개의 메시지로 분할되어 수신될 수 있다.

### 처리 방식

- 수신된 메시지 배열의 body를 하나의 문자열로 결합
- 결합된 메시지를 기준으로 키워드 검사 및 전달 수행

---

## 6. 중복 전달 방지

BroadcastReceiver는 환경에 따라 중복 호출될 수 있다.

### 방지 방법

- 메시지 식별값 생성
  - sender + body + timestamp 기반 hash
- 최근 처리 메시지 캐시 저장
- 동일 메시지 재처리 방지

---

## 7. 전달 루프 방지

전달 대상 번호가 감시 번호와 동일한 경우 무한 전달 가능성이 존재한다.

### 방지 조건

- 전달 대상 번호가 규칙의 발신 번호와 동일한 경우 전달 차단
- 자기 자신에게 전달 금지

---

## 8. Manifest 설정 보완

### receiver 보안 설정

```xml
android:exported="false"
````

외부 앱에 의한 호출 차단.

### priority 설정 제거

Android 8 이상에서 Broadcast priority는 실질적 영향이 없으므로 제거 권장.

---

## 9. 사용자 환경 안내 (권장)

앱 설정 화면 또는 최초 실행 시 안내 문구 제공.

### 안내 내용

* 일부 기기에서 SMS 자동 처리 제한 가능
* 배터리 최적화 해제 권장
* 기본 SMS 앱 설정 시 안정성 향상 가능

현재 구조는 단순 구현에 적합하나 기능 확장 시 다음 구조 권장.
